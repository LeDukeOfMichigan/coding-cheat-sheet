HTML Tips

Tags to document:

HTML Tag Self-closing? Description
<a> No This creates a hyperlink based on the href attribute
<div> No This is used to style an area within a webpage, but should only be used when semantic elements do not apply. Note: By default, browsers always place a line break before and after the <div> element.
<span> No Similar to the <div> element, this can be used to style text, but <span> is used inline while <div> is used at the block level.
<article> No The <article> HTML element represents a self-contained composition in a document, page, application, or site, which is intended to be independently distributable or reusable (e.g., in syndication).
<aside> No This is a portion of a document whose content is indirectly related to the main content. These are often used for sidebars and call-out boxes.
<section> No This is a generic standalone section of a document, which doesn't have a more specific semantic element that can represent it. These should always contain a heading, and usually a paragraph as well.
<header> No Distinct from the <head> element, a header will define the top portion of a document. This is frequently used for site-level navigation and the title of the site.
<nav> No This specifies a section to provide hyperlinks to other parts of the site or other resources. These can be menus, tables of contents, or indexes.
<video> No This will load a video file into the webpage and specify what controls are available to the user. Content inside this tag is only referenced when browsers don't support the video element. The <source> element is needed to define where the video file is stored.
<script> No This inserts a block of code that can contain JavaScript. This can be used to make the site more interactive.
<img> Yes This adds an image to the page. The src attribute is needed to define the location of the image.
<br> Yes This creates a line break in the text (carriage-return). This should not be used to separate paragraphs.
<input> Yes This is used to create interactive controls for web-based forms to accept user data. Many different input types are available.
<source> Yes This is used in the context of a <picture>, <audio>, or <video> element to define the type of file and its location using the src attribute.
<param> Yes Used within an <object> element to define specific parameters for the object.
<meta> Yes Used in the <head> element to define site elements that are machine-readable, but not displayed.
<link> Yes Used in the <head> element to define a relationship with an external resource.
